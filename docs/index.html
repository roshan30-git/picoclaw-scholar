Core Capabilities

Designed for retention.
Built for exams.

StudyClaw replaces passive reading with active recall, spaced repetition, and 5-year pattern prediction.

Contextual Ingestion

Forward PDFs directly inside WhatsApp. The Indexer Agent parses text, OCRs handwritten notes, and extracts formulas using Flash Vision to silently build your personal knowledge base.

5-Year PYQ Predictor

The Strategist Agent analyzes Structural Pattern Frequency of past papers. It predicts the most likely conceptual topics for your upcoming exams and ranks them by probability percentage.

Adaptive Quizzing

StudyClaw pushes daily quizzes mapped exclusively to your syllabus. It profiles your learning pace and automatically generates Sunday flashcards capturing your weakest topics.

On-Demand Visuals

Sometimes text isn't enough. When you ask about complex systems, StudyClaw natively generates Mermaid flowcharts, SVG electrical circuits, and KaTeX mathematical formulas rendered inside a lightweight local viewer.

Architecture Documentation

Local orchestration.
Cloud reasoning.

StudyClaw is a highly structured Go application running a multi-agent loop locally on your device (natively supported on Android via Termux). It is not a thin wrapper.

Message ingestion, OCR bounding, and context retrieval occur inside a local SQLite database. It calls the Google Gemini API strictly for reasoning tasks, maintaining extreme token efficiency by routing simple NLP via `gemini-2.0-flash-lite` and complex execution to `gemini-2.5-pro`.

  • 01 Pub/Sub Event Bus for decoupled system agents
  • 02 Zero-API Persona Router (8 localized agent overlays)
  • 03 SQLite Schema specifically tuned for vector embeddings
  • 04 WhatsApp integration via Baileys API/Go bridge
pkg/agent/loop.go
func (l *AgentLoop) handleMessage(ctx context.Context, msg Message) {
    // 1. Determine Intent Locally (No API calls)
    persona := l.router.Classify(msg.Content)
    
    // 2. Build Context
    history := l.db.GetHistory(msg.Sender, 5)
    refs := l.reflectionMgr.GetRecentReflections(3)
    
    // 3. Tool Calling Execution Loop
    tools := l.getToolDefinitions(persona)
    resp, err := l.provider.Chat(ctx, history, tools)
    
    // 4. Render Visual Tags (, )
    renderedMsg := l.visParser.ApplyVisuals(resp)
    
    l.bus.Publish(renderedMsg)
}
